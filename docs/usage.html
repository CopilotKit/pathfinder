Getting Started

Two install paths. Same server. Choose what fits your workflow.

Quick Start

CLI

$ npx @copilotkit/pathfinder init $ npx @copilotkit/pathfinder serve

Quickest start. No Docker needed. Add Postgres later for semantic search.

Docker

$ docker pull ghcr.io/copilotkit/pathfinder $ docker run -v ./pathfinder.yaml:/app/pathfinder.yaml \ -v ./docs:/app/docs -p 3001:3001 \ ghcr.io/copilotkit/pathfinder

Production-ready. Use docker-compose to add Postgres for full semantic search.

Configuration

Pathfinder is configured via a single pathfinder.yaml file. Here is an annotated example:

server: name: my-project-docs sources: - name: docs type: markdown repo: https://github.com/your-org/your-repo.git path: docs/ file_patterns: ["**/*.mdx", "**/*.md"] chunk: target_tokens: 600 overlap_tokens: 50 tools: # Bash tools work without any database or API keys - name: explore-docs type: bash sources: [docs] bash: session_state: true grep_strategy: passthrough # change to hybrid or vector for semantic grep # Search tools require Postgres + OpenAI embeddings - name: search-docs type: search source: docs default_limit: 5

Bash tools (find, grep, cat, ls, head) work immediately with no external dependencies. Search tools require a Postgres database with pgvector and an OpenAI API key for computing embeddings.

Adding Semantic Search

To enable search tools and vector-backed grep, add Postgres and an OpenAI API key:

1. Set environment variables

DATABASE_URL=postgresql://user:pass@localhost:5432/pathfinder OPENAI_API_KEY=sk-...

2. Add embedding config to your YAML

embedding: model: text-embedding-3-small dimensions: 1536

3. Add a search tool

tools: - name: search-docs type: search source: docs

4. Use docker-compose for the full stack

# docker-compose.yml services: pathfinder: image: ghcr.io/copilotkit/pathfinder ports: - "3001:3001" volumes: - ./pathfinder.yaml:/app/pathfinder.yaml - ./docs:/app/docs environment: - DATABASE_URL=postgresql://pathfinder:pathfinder@postgres:5432/pathfinder - OPENAI_API_KEY=${OPENAI_API_KEY} depends_on: - postgres postgres: image: pgvector/pgvector:pg16 environment: POSTGRES_USER: pathfinder POSTGRES_PASSWORD: pathfinder POSTGRES_DB: pathfinder volumes: - pgdata:/var/lib/postgresql/data volumes: pgdata:

MCP Client Setup

Point your AI agent's MCP configuration at Pathfinder. The server exposes a single HTTP endpoint.

Claude Desktop

Add to ~/.claude/mcp.json:

{ "mcpServers": { "docs": { "type": "http", "url": "http://localhost:3001/mcp" } } }

Cursor

Add to .cursor/mcp.json:

{ "mcpServers": { "docs": { "type": "http", "url": "http://localhost:3001/mcp" } } }

qmd

When grep_strategy is set to vector or hybrid in your bash tool config, Pathfinder exposes a qmd command inside the bash sandbox. This is a semantic search command that agents can use alongside regular shell tools.

$ qmd "how do I handle errors" /docs/guides/error-handling.mdx (0.94) /docs/reference/hooks.mdx (0.87) /docs/guides/streaming.mdx (0.81)

qmd searches your embedded docs using vector similarity, returning ranked results with relevance scores. Regular grep passes through to real bash unchanged -- it is never intercepted or replaced.

When an agent runs grep and gets no results, Pathfinder appends a hint suggesting qmd as an alternative. This nudges agents toward semantic search when exact-match grep misses, without forcing a workflow change.
